Overview

What this website is

This repository powers Palmer Gill's personal project website. The public root page is a project launcher, and each project lives in its own static folder. Shared API features run through a FastAPI backend in backend/.

Frontend Static HTML, CSS, and JavaScript

Each app folder contains its own browser entrypoint and assets.

Backend FastAPI

Shared routes support stock data, poker, Bitcoin chat, and admin logs.

Hosting Vercel + Railway

Static routes stay on the website host; /api/* rewrites to Railway.

Local Development

Run the site locally

Use the repo-level startup script to run the FastAPI backend and locally mounted static pages together.

./start.sh

Then open:

http://127.0.0.1:8000

Protected local app routes and API routes use Basic Auth when credentials are provided. Poker, craps, blackjack static hosting, and /api/poker/* are intended to stay public. 

APP_AUTH_USERNAME=palmer APP_AUTH_PASSWORD=your-password ./start.sh

Backend logs are written to:

logs/backend.log

The shared backend still reserves /docs for generated FastAPI API docs when served directly. Both backend API docs and the static website docs route should stay behind Basic Auth.

Architecture

How requests move through the system

Browser
static pages
Static host
/api/*
Railway FastAPI
service clients
Market data, Bitcoin RPC, database cache, logs

In local development, LOCAL_SITE_ROOT=true lets FastAPI serve the root site and active app folders from the same process. In production, the public website is static and the API backend is treated as a separate service.

Deployment

Production model

  • Static hosting serves the repo root and active app directories.
  • Vercel rewrites /api/* to the Railway backend.
  • Railway builds from the root Dockerfile and runs backend/app/main.py.
  • The root Dockerfile copies backend/; the standalone poker/backend/ service is retained for reference.

The Railway runtime command is:

uvicorn app.main:app --host 0.0.0.0 --port ${PORT:-8000}

Primary deployment variables:

APP_AUTH_USERNAME APP_AUTH_PASSWORD USE_REAL_DATA POLYGON_API_KEY FINNHUB_API_KEY BITCOIN_RPC_URL BITCOIN_RPC_USER BITCOIN_RPC_PASSWORD OPENAI_API_KEY

Authentication

Protected and public routes

App authentication uses HTTP Basic Auth. If APP_AUTH_PASSWORD is missing in production, protected routes return 503 instead of accidentally publishing private tools.

Public

  • /
  • /poker/*
  • /craps/*
  • /blackjack/*
  • /api/poker/*

Protected

  • /docs/*
  • /stock-research/*
  • /bitcoin-chat/*
  • /admin/*
  • /api/* except /api/poker/*

Backend API

Shared FastAPI routes

/api/stocks/*

Stock lookup, summaries, earnings, fundamentals, and price history for the stock research app.

/api/poker/*

Integrated poker game endpoints for game creation, game state, player actions, buy-back, next hand, and health.

/api/bitcoin/*

Bitcoin health, node status, latest block, block lookup, transaction lookup, mempool summary, and chat.

/api/admin/*

Protected structured log and file-tail endpoints used by the admin dashboard.

/health

Backend health check endpoint.

/docs

Generated FastAPI API documentation when talking directly to the backend service.

Project Docs

App-specific documentation

Stock Research

Uses Polygon as the primary market data source, with optional Finnhub earnings estimates. The backend derives ratios such as P/E, revenue growth, ROE, ROA, ROIC, debt-to-equity, margins, free cash flow, and balance-sheet metrics from provider data.

Texas Hold'em Poker

Browser poker app with AI opponents, multiplayer lobbies, persisted player stats, theme controls, card deck themes, dark mode, PWA support, and a public integrated API.

Open app

Craps

Single-player dice table with Pass Line, Come, and proposition betting. The game handles point state, payouts, and dice results in the browser.

Open app

Blackjack

Single-player blackjack with a six-deck shoe, dealer hits soft 17, 3:2 naturals, double after split, pair splitting, split aces, insurance, and focused game-rule tests.

Open app

Bitcoin Chat

Chat interface connected to read-only Bitcoin Core RPC methods through Railway and a Cloudflare Tunnel. It can answer questions about blocks, transactions, mempool fees, node status, and mined-BTC data.

Open app

Admin

Protected dashboard for backend logs and operational inspection. Keep it behind Basic Auth in both Vercel and Railway environments.

Data Providers

External services

Polygon.io

Primary Stock Research data source.

  • Ticker search: /v3/reference/tickers
  • Ticker details: /v3/reference/tickers/{ticker}
  • Previous close: /v2/aggs/ticker/{ticker}/prev
  • Historical prices: /v2/aggs/ticker/{ticker}/range/*
  • Financial statements: /vX/reference/financials

Bitcoin Core RPC

Read-only Bitcoin node data for Bitcoin Chat.

  • Production tunnel hostname: bitcoin-rpc.palmergill.com
  • Allowed methods include block, transaction, mempool, and fee-estimate reads.
  • Wallet, signing, broadcast, and private-key methods should stay disabled.
